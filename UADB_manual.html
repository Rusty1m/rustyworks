UADB 取説 / Manual

UADB (Universal Asset Database) is an editor extension for browsing, searching, tagging, and maintaining project assets.

概要 / Overview

Japanese

UADB は、1つのウィンドウでアセット一覧、検索、タグ管理、プロパティ確認を行うためのツールです。 元ファイルを勝手に移動・改名しない、非破壊型の運用を前提にしています。

軽い操作感、高速な一覧切り替え、日常的に開きやすい使い心地を重視しています。

English

UADB is a tool for browsing assets, searching, managing tags, and checking properties in one window. It is designed for non-destructive workflow and does not normally move or rename your project files.

It focuses on lightweight operation, fast view switching, and a comfortable day-to-day workflow.

クイックスタート / Quick Start

Japanese

  1. Window > UADB を開きます。
  2. 上部の 更新 を押し、リストを更新 を選びます。
  3. 必要なら左側の検索欄にキーワードを入力します。
  4. 結果をクリックして、右側のプロパティを確認します。
  5. タグ編集や一括処理が必要なら、チェックボックスと左側パネルを使います。

English

  1. Open Window > UADB.
  2. Press Rescan and choose Refresh List.
  3. If needed, type a keyword in the search field.
  4. Click a result and check the property panel on the right.
  5. For tag editing or bulk actions, use checkboxes and the left-side action panel.
初回はまず リストを更新 / Refresh List から始めるのが基本です。
For first use, start with Refresh List.

画面概要 / Window Overview

上部バー / Top Bar

  • UADB: メインメニュー / Main menu
  • 更新 / Rescan: リストを更新 / Refresh List と 未管理ファイルの読み込み / Load Unmanaged Files
  • Undo / Redo: UADB 内の操作を戻す / やり直す
  • 削除 / Delete: チェック済み項目を削除。Undo 可能
  • ステータス: 管理対象数と未管理 / 新規数を表示

左パネル / Left Panel

  • 検索 / Search: キーワード検索
  • 検索範囲 / Scope: 全体、タグ、表示名、呼び出し名、フルパス
  • 表示切り替え / View Mode: リスト、アイコン、プレビュー
  • 表示をリセット / Reset View: 検索と表示状態を初期化
  • タグを追加 / 置換え: 一括処理

中央ペイン / Center Pane

  • 検索結果を一覧またはグリッドで表示
  • 列: 呼び出し名、表示名、タグ、表示項目
  • チェックボックスで複数選択
  • 単クリックは UADB 内選択、ダブルクリックはエディタ側の選択連動

右パネル / Right Panel

  • プロパティ / Properties: 表示名、キー、タグ、JSON
  • プレビュー / Preview: 選択項目の静的プレビュー
  • タグリスト / Tag List: タグから絞り込み
  • 設定 / Settings: 言語、保存方式、管理対象フォルダなど

基本的な使い方 / Workflow

1. リストを更新 / Refresh the List

上部の 更新 を押して リストを更新 を選ぶと、現在のプロジェクト状態に合わせて一覧を再構築します。

Press Rescan and choose Refresh List to rebuild the current list from project assets.

2. 検索する / Search

左上の検索欄にキーワードを入れ、必要に応じて 検索範囲 / Scope を切り替えます。

Type a keyword and change Scope when needed.

3. 表示を切り替える / Change the View

リスト / アイコン / プレビュー を切り替えて見やすい形式で探します。

Switch between List / Icon / Preview to browse in the most useful way.

4. プロパティを編集する / Edit Properties

項目を選択すると右側に詳細が出ます。表示名、キー、タグ、データ項目なら JSON を編集できます。

Select an item to edit display name, key, tags, and JSON for data entries.

5. 一括処理 / Bulk Actions

チェックボックスで複数項目を選び、左側の タグを追加 / 置換え から一括処理します。

Use checkboxes and the bulk action panel for tag add or replace actions.

6. 未管理ファイルの読み込み / Load Unmanaged Files

上部の 更新 メニューから 未管理ファイルの読み込み を選ぶと、まだ UADB に入っていない対象を追加できます。

Choose Load Unmanaged Files from the Rescan menu to add files that are not yet in UADB.

復旧ガイド / Recovery Guide

Japanese

  • 表示復旧: 一覧、選択、プレビューの接続をやり直します
  • セーフモード: アイコン / プレビューを止めてリスト表示に戻します

白化や一時的な表示崩れが起きた時に使います。

English

  • Recover View: reconnects list, selection, and preview state
  • Safe Mode: disables icon / preview rendering and falls back to list-only mode

Use these when the window becomes blank or the visual state is broken.

注意事項 / Notes

Japanese

  • 通常動作では任意のプロジェクトアセットを移動・改名・物理削除しません
  • 削除した項目は通常一覧から外れますが、Undo / Redo で追跡できます
  • 未管理ファイルの読み込みは、まだ UADB に入っていない項目を追加する処理です

English

  • Normal use does not move, rename, or physically delete arbitrary project assets
  • Deleted entries leave the normal list view, but can still be tracked with Undo / Redo
  • Load Unmanaged Files adds items that are not yet stored in UADB

FAQ

Q. 「更新」と「未管理ファイルの読み込み」は同じですか？

いいえ。リストを更新 は現在の一覧を再同期する処理です。未管理ファイルの読み込み は、まだ UADB に入っていない対象を新規追加します。

No. Refresh List refreshes the current list. Load Unmanaged Files adds files that are not yet in UADB.

Q. 単クリックとダブルクリックの違いは？

単クリックは UADB 内の選択更新です。ダブルクリックはエディタ側の選択や Inspector 連動に使います。

Single click updates selection inside UADB. Double click syncs selection with the editor and Inspector.

Q. タグだけで絞れますか？

はい。検索範囲を タグ / Tag に切り替えると、タグだけを対象に検索できます。

Yes. Set the scope to Tag to search tag values only.

Q. 逆参照は重いですか？

プロジェクト規模によっては少し時間がかかることがあります。必要な時だけ確認する使い方をおすすめします。

It can take some time on large projects. Use it when you need it, not constantly.
